Entropy encoders

Turning quantized symbols and run-length tuples into compact bitstreams.

Huffman (canonical backend)

Our Huffman path builds a frequency table over the exact symbols we will encode (DPCM deltas for DC, RLE tuples for AC), trains a codec from those counts, and writes raw compressed bytes. The hybrid pipeline stores separate DC and AC streams plus their symbol count tables in the file container so the decoder can reconstruct both Huffman trees.

ANS & arithmetic variants

Additional pipelines experiment with asymmetric numeral systems (ANS) and static arithmetic coding on the same RLE tuple vocabulary. ANS flattens tuples to a symbol stream and packs state plus code words into a custom blob; arithmetic coding maps stringified value_count symbols through a scaled static model.

Encoder-specific packing lives in the pipeline scripts; the on-disk container only holds metadata and named bitstreams blobs so each backend stays swappable.
