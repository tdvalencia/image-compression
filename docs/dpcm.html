DPCM on DC coefficients

Differential pulse-code modulation reduces redundancy in the DC sequence before Huffman coding.

Goal

Each 8×8 block has one DC DCT coefficient that tracks coarse brightness. Neighboring blocks in natural images tend to have similar DC values, so the raw DC sequence is spatially smooth. The AC coefficients are already sparse and zig-zag friendly for RLE; the DC path benefits more from prediction.

Math

We apply first-order DPCM to the DC sequence: keep the first DC as an anchor and encode the rest as deltas: \(d_0 = x_0\), \(d_i = x_i - x_{i-1}\). Decoding reconstructs by cumulative sum: \(x_0 = d_0\), \(x_i = x_{i-1} + d_i\).

This works well because DC values are strongly correlated across adjacent blocks, so the delta histogram is tighter and entropy coding is more efficient.

Code

In the hybrid pipeline (codec/pipelines/hybrid_hf_compression.py), we extract DC coefficients during zig-zag serialization and then call dpcm.apply_dpcm from codec/encoders/dpcm.py.

On decode we call dpcm.invert_dpcm to rebuild the raw DC stream before recombining it with decoded AC coefficients and running inverse quantization and inverse DCT.

Hand-off

Output is a sequence of DC deltas. Next stage is entropy coding of both streams: Huffman (or variants) encodes DC deltas and AC run-length symbols into bitstreams (see entropy encoders).
