Quantization

Where we actually throw information away: after the transform, before entropy coding.

Goal

Turn continuous-valued transform coefficients into small integers so entropy coding can work well. Quantization is the main lossy step: it controls the rate–distortion tradeoff by forcing high-frequency coefficients toward zero.

Math

We use an 8×8 quantization matrix \(Q\) whose entries increase with spatial frequency. Quantization is: \( \hat{C}_{i,j} = \text{round}(C_{i,j} / Q_{i,j}) \), and dequantization is \( \tilde{C}_{i,j} = \hat{C}_{i,j} \cdot Q_{i,j} \).

In the hybrid pipeline we apply different effective step sizes per PCA component: the first component uses the base matrix, and the second and third components use a scaled matrix (e.g. \(6\times\)) so those channels are quantized more coarsely.

Code

The base matrix is built by build_quantization_matrix in codec/tools.py. In codec/pipelines/hybrid_hf_compression.py we divide DCT blocks by that matrix and round to int32, using heavier scaling for PCA channels 2 and 3.

The decoder rebuilds the same matrix and multiplies to dequantize before inverse DCT, so encoder and decoder stay matched aside from rounding loss.

Hand-off

Output is an integer-valued block tensor. Next we serialize each block in zig-zag order, split DC and AC streams, run DPCM on the DC stream, and entropy encode both streams (see entropy encoders).
