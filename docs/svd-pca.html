SVD & PCA

Low-rank spatial models versus a color-space rotation ahead of the DCT.

Goal

Make the image easier to compress before we start blocking and transforming. In our main codec we use PCA as a color decorrelation front-end so most energy concentrates in the first component and the remaining components can be quantized more aggressively.

We also explored SVD as a possible low-rank spatial model, but it is not the primary representation in the hybrid Huffman pipeline.

Math

PCA (used in the hybrid pipeline)

Treat each pixel as a 3-vector \(x \in \mathbb{R}^3\). PCA finds an orthonormal basis that decorrelates RGB by rotating into “eigen-channels”. In practice we store the rotation and the mean so the decoder can invert the transform.

Encoder: \(y = V^T (x - \mu)\). Decoder: \(x \approx V y + \mu\).

SVD (exploration)

For a single 2D channel matrix \(A\), SVD writes \(A = U \Sigma V^T\). Truncating small singular values yields a rank-\(k\) approximation that can reduce the number of degrees of freedom, but it is computationally heavier and needs careful quantization/packing of factors for a practical codec.

Code

The hybrid pipeline performs PCA as its first step: codec/pipelines/hybrid_hf_compression.py calls pca.apply_pca_prefilter. The implementation lives in codec/encoders/pca.py.

Decoder-side metadata for PCA (stored in the container) includes pca_Vt and pca_means so reconstruction can invert the rotation.

Hand-off

Output is a 3-channel PCA-rotated image in floating point (plus metadata: Vᵀ and means). Next stage is blocking + DCT, followed by quantization with heavier scaling on the second and third PCA components.
